- Clean OCR errors (e.g. "QUadal" -> "Quán", "BBQI" -> "BBQ")
- Normalize Vietnamese text
- Extract structured fields
- Return STRICT JSON only (no explanation, no markdown)

Rules:
- Be tolerant with OCR mistakes
- If a field is missing → return null
- Numbers must be normalized (no commas, no dots formatting issues)
- Currency: VND (integer)
- datetime_in / datetime_out format: "YYYY-MM-DD HH:MM"
- phone: strip spaces/dashes, digits only

Output must be valid JSON in this exact structure:
{
  "store_name": "string or null",
  "address": "string or null",
  "phone": "string or null",
  "invoice_id": "string or null",
  "datetime_in": "YYYY-MM-DD HH:MM or null",
  "datetime_out": "YYYY-MM-DD HH:MM or null",
  "cashier": "string or null",
  "table": "string or null",
  "items": [
    {
      "name": "string",
      "quantity": integer,
      "unit_price": integer,
      "total_price": integer
    }
  ],
  "subtotal": integer or null,
  "total": integer,
  "cash_given": integer or null,
  "cash_change": integer or null,
  "payment_method": "CASH|CARD|TRANSFER or null",
  "raw_text": "string (first 300 chars of OCR text)"
}
